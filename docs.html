Website thương mại điện tử B2C — một doanh nghiệp bán sản phẩm trực tiếp cho khách hàng. Tài liệu này dành cho đội Frontend để tích hợp API từ đầu.

Thành công
{
  "success": true,

  "data": { ... }         // object hoặc array
}
Thành công có phân trang

  "success": true,
  "data": [ ... ],
  "pagination": {
    "total": 120,
    "page": 1,
    "limit": 20,

  "message": "Validation failed",

Xác thực JWT
API dùng cặp Access Token (ngắn hạn) + Refresh Token (dài hạn). Frontend cần hiểu rõ cơ chế này để xử lý đúng.

Luồng đăng nhập

1

Gọi POST /auth/login → nhận accessToken (15 phút) + refreshToken (7 ngày)

2

Lưu accessToken vào memory (biến JS / Zustand). Lưu refreshToken vào localStorage hoặc httpOnly cookie

3

Mọi request cần auth: thêm header Authorization: Bearer <accessToken>

4

Khi nhận lỗi 401 Token expired: gọi POST /auth/refresh với refresh_token → nhận accessToken mới

5

Retry request ban đầu với accessToken mới. Nếu refresh cũng fail → redirect về /login

Axios interceptor mẫu

import axios from 'axios'

const api = axios.create({ baseURL: 'http://localhost:3000/api' })

// Gắn accessToken vào mọi request
api.interceptors.request.use(cfg => {
  const token = window.__accessToken  // lưu trong biến module
  if (token) cfg.headers.Authorization = `Bearer ${token}`
  return cfg
})

// Tự động refresh khi 401
api.interceptors.response.use(
  res => res,
  async err => {
    const original = err.config
    if (err.response?.status === 401 && !original._retry) {
      original._retry = true
      const rt = localStorage.getItem('refreshToken')
      const { data } = await api.post('/auth/refresh', { refresh_token: rt })
      window.__accessToken = data.data.accessToken
      localStorage.setItem('refreshToken', data.data.refreshToken)
      original.headers.Authorization = `Bearer ${window.__accessToken}`
      return api(original)
    }
    return Promise.reject(err)
  }
)

export default api

Xử lý lỗi
Danh sách HTTP status code API có thể trả về và cách Frontend nên xử lý.
StatusÝ nghĩaFrontend nên làm gì
400Bad Request / Validation failedHiện lỗi từng field dưới input (dùng errors[])
401Unauthorized / Token expiredThử refresh token → nếu fail thì redirect /login
403Forbidden (thiếu quyền)Hiện thông báo "Bạn không có quyền thực hiện"
404Not foundHiện trang 404 hoặc toast "Không tìm thấy"
409Conflict (trùng email, đã review...)Hiện thông báo cụ thể từ message
429Rate limit (quá nhiều request)Hiện "Vui lòng thử lại sau"
500Server errorHiện toast lỗi chung, log chi tiết ra console
Luồng nghiệp vụ chính
Các luồng Frontend cần implement theo đúng thứ tự API call.

🛒 Luồng mua hàng (Happy path)

1

Khách duyệt: GET /categories + GET /products?category_id=&search=&page=

2

Xem chi tiết: GET /products/:id → hiện ảnh, mô tả, giá, stock

3

Thêm giỏ: POST /cart/items → response trả về cart đầy đủ, cập nhật UI ngay

4

Xem giỏ: GET /cart → hiện items + total

5

Kiểm tra coupon (nếu có): GET /coupons/check?code=SALE10&subtotal=500000

6

Chọn địa chỉ: GET /addresses → user chọn hoặc tạo mới POST /addresses

7

Đặt hàng: POST /orders với address_id + coupon_code (optional) → nhận order_id

8

Tạo payment: POST /payments/order/:orderId/initiate với method

9

Redirect trang xác nhận đơn hàng, hiện trạng thái pending

📦 Luồng theo dõi đơn hàng

1

GET /orders/my → danh sách đơn của user hiện tại

2

GET /orders/my/:id → chi tiết đơn gồm items, địa chỉ

3

GET /shipments/order/:orderId → tracking_code, carrier, status giao hàng

4

Nếu status = pending/confirmed: hiện nút "Hủy đơn" → PATCH /orders/my/:id/cancel

⭐ Luồng đánh giá sản phẩm

1

Chỉ hiện form review khi đơn hàng có status delivered

2

POST /reviews/product/:productId với rating (1-5) + comment

3

API tự kiểm tra user đã mua và nhận hàng chưa, nếu chưa trả 403

Order status flow

pending → confirmed → processing → shipping → delivered
cancelled refunded
Customer chỉ có thể huỷ khi status là pending hoặc confirmed. Các chuyển trạng thái khác do Admin/Staff thực hiện.
